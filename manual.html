Structure

페이지 구성

메인 랜딩

홈, 후보 소개, 명곡·봉림 소통지도까지 첫 화면 흐름에 배치했습니다.

햄버거 메뉴

홈, 소개, 지도, 공약, 동네, 활동, 문의 페이지로 이동합니다.

분리 페이지

/policies/, /district/, /activities/, /contact/가 별도 화면으로 제공됩니다.

매뉴얼

현재 문서는 /manual.html에서 확인할 수 있는 운영자용 안내 페이지입니다.

Features

주요 기능

1. 메인 화면

  • PC와 모바일에 각각 최적화된 후보 메인 이미지를 사용합니다.
  • 지역구, 인구, 세대 정보를 첫 화면에 노출합니다.
  • 헤더에는 후보명, 기호, 더불어민주당 심볼, 햄버거 메뉴가 들어갑니다.

2. 유튜브 팝업

  • 홈페이지 접속 시 로고송 유튜브 영상 팝업이 열립니다.
  • 브라우저 정책 때문에 자동재생은 음소거 상태로 시도됩니다.
  • 오늘 하루 보지 않기를 누르면 해당 날짜에는 다시 뜨지 않습니다.

3. 네이버 지도 기반 소통지도

  • 지역거점, 공약, 활동 필터로 지도 핀을 분류합니다.
  • 핀 선택 시 제목, 지역, 좌표, 설명이 함께 표시됩니다.
  • 네이버 지도 API 서비스 환경에는 운영 도메인과 개발 도메인이 등록되어야 합니다.

4. 공약 페이지

  • 유튜브에서 확인한 공약 영상을 기준으로 카드형 공약을 구성했습니다.
  • 각 공약 카드에는 공약명, 분야, 요약, 상태 문구, 영상 링크가 들어갑니다.
  • 현재 반영된 공약은 포켓 주차장, 상생협의체, 제2외국어, 세금 감시, 불법 홍보관 퇴출입니다.

5. 공유 썸네일

  • 카카오톡 공유용 이미지는 /images/share-kakao.png를 사용합니다.
  • 문자·일반 공유용 이미지는 /images/share-message.png를 사용합니다.
  • 카카오톡 미리보기가 바로 바뀌지 않으면 카카오 공유 디버거에서 캐시를 갱신해야 합니다.
Operation

운영 방법

콘텐츠 수정 위치

  • 후보명, 슬로건, 공약, 지도 핀, 인스타그램 주소는 src/data/candidate.ts에서 관리합니다.
  • 이미지는 public/images/에 넣은 뒤 빌드하면 배포 루트의 images/로 복사됩니다.
  • 푸터 저작권은 짓마케팅, 대표 서효승으로 표기되어 있습니다.

배포 흐름

  1. 소스 수정 후 npm run build로 정적 파일을 생성합니다.
  2. out 폴더의 결과물을 저장소 루트로 복사합니다.
  3. GitHub에 커밋·푸시하면 GitHub Pages가 반영합니다.
  4. 커스텀 도메인 류성국.kr은 Pages 반영 후 몇 분 정도 캐시가 남을 수 있습니다.

도메인, 네이버 지도 API, 카카오 공유 썸네일처럼 외부 서비스가 연결된 기능은 배포 후 실제 도메인에서 한 번씩 확인하는 것이 좋습니다.

Checklist

수정 후 확인 체크리스트

  • PC와 모바일에서 헤더가 겹치지 않는지 확인합니다.
  • 홈 첫 화면에서 후보 이미지와 통계 카드가 자연스럽게 보이는지 확인합니다.
  • 유튜브 팝업의 닫기, 오늘 하루 보지 않기 버튼이 작동하는지 확인합니다.
  • 햄버거 메뉴의 모든 링크가 정상 이동하는지 확인합니다.
  • 네이버 지도 핀이 표시되고 필터 버튼이 작동하는지 확인합니다.
  • 카카오톡 공유 시 제목, 설명, 썸네일이 의도대로 보이는지 확인합니다.
  • 공약 영상 링크가 새 창으로 열리는지 확인합니다.
